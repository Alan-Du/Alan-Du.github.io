Working Sample & Projects

Below are some Quant-Finance projects I have done since 2018. They are mainly in interests rate modeling and pricing includes curve building, Derivatives(Swaps/FXs/Options) pricing, XVA and VaR. They are all built in PYTHON with QuantLib as support library. I will walk you through all basic modules in Fixed income area with a focus on rates trading from scratch.
NOTE: All examples and codes here are open to everyone, please feel free to download or modify for personal using. Also, all examples and codes here will be in HTML format with JupyterNoteBook output. If you find any bugs/errors or need some helps with implementing please email me.

To run all projects below, you would need a scientific Python environment and QuantLib-Python installed in your system. Although you can install each of the required libraries manually, the simplest way to get them all is to download and install: Anaconda Python (or any other scientific Python distribution).

  1. In this section, I will start with curve building method in Fixed Income area. The importance of curve building sets the backbone for almost all analyze staff in Fixed Income area. From where you might be able to learn and do analysis of Risks, Valuations and Simulations. Basically there are three different curves involved: LIBOR curve, FX curve and OIS curve.
    NoteBook:BootStrapping.html
    Download:RightClick->Save

  2. In this section, I will talk about a directly applied area with bootstrapped curves. This could be consider the most fundamental application and daily used function in Fixed Income area. In the below, I will detail pricing on swaps with LIBOR curve, FX with FX curve (currency collateral adjusted curve) and Basis swap with OIS curve (Overnight rates curve).
    NoteBook:SwapPricing.html
    Download:RightClick->Save

  3. In this section, we will be using all knowledge leaned above and apply with another level of complexity. Pricing of options in interests rate area involved curve building and basic derivatives pricing. Moreover, we will start to introduce another structure, the volatility surface (for cap/floor) volatility curve (for swaption). In this section, we only focus on applying Black formula with current market instruments data to price. However, in the next section, I will talk more about how to use more advanced mathematical method called Monte-Carlo simulation to solve the same problem.
    NoteBook:BasicOptionPricing.html
    Download:RightClick->Save

  4. In this section, I will walk you through Short-Rate model for rate curve generation and simulation. This is a prerequisite for Monte-Carlo simulation method for more advanced option pricing theory and application. Since nowadays, more and more options are not exotic basic black formula will not help for this type of pricing. Instead of waiting mathematician developing new analytic formula people prefer just run all possible situations and take the average. In order to run pricing on different curve scenario, we need a short-rate model generation engine to build curves.
    NoteBook:ShortRateModel.html
    Download:RightClick->Save

  5. In this section, I will walk you through how to use Short-Rate model and swap pricing method to calculate CVA/DVA. This section depends heavily on my previous posts like "Bootstrapping", "SwapPricing" and "ShortRate Model". Since financial crisis 2008, investors pays more and more attention on credit risks and the CVA evolvs into a bigger topic in pricing adjustments. Bascially, an X-Value Adjustment (XVA) is a generic term referring collectively to a number of different “Valuation Adjustments” in relation to derivative instruments held by banks. The purpose of these is to hedge for possible losses due to counterparty default; but also, to determine (and hedge) the amount of capital required under Basel III. For a discussion as to the impact of xVA on the bank's overall balance sheet, return on equity, and dividend policy, in many institutions, led to the creation of specialized desks.
    NoteBook:XVACalculation.html
    Download:RightClick->Save

  6. In this section, I will apply Monte-Carlo method to another interesting topic: VaR calcualtion. ALso will talk about how to use duration method as well as stresstesting on swaps. Basically, we are applying monte-carlo gieven a time window and try to simulate yield curves in a future time. After we get a set of termstructures, we re-calculate swap values and calculate corresponding quantile for VaR. The method for stresstesting is simply reused what I have listed in SwapPricing notebook plus user defined of stressed scenarios. For detailed information, PLEASE review my previous posts: "SwapPricing" and "ShortRateModel".
    NoteBook:VaR_StressTesting.html
    Download:RightClick->Save
